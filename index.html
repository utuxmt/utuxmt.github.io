CS224 Lecture3 筆記 | Word Vector Representation (2)

Posted on 2018-06-18

    這堂課會延續上一堂的內容繼續討論 Word Vector，包含原先 Skip-Gram 模型的缺點、改進的方式以及其他 Word Vector 的模型。影片在這，投影片在這。

Nagative Sampling

    在前一堂課我們學到了當文本共有 $V$ 個字彙，$o$ 為 $c$ 的 context word 的機率為
$$ P(o|c) = \frac{exp(\mu_{o}^{T}\nu_{c})}{\sum_{w \in V}exp(\mu_{w}^{T}\nu_{c})} \tag{1} $$
這個式子需要計算 文本長度 T $\times$ window size 次，想像隨著文本大小的增加，需要的計算量會有多驚人。所以後來 Mikolov et al. 2013 就提出了 Nagative Sampling 的改進方式。

Intuition

    Nagative Sampling 基本的想法是：「與其用所有的字彙來計算機率，不如自己隨機選擇一些不會是 context word 的字彙，讓模型學習去分辨輸入的字彙是否為 context word」。舉例來說，我的文本裡有 I like deep learning and NLP. 這個句子，當 context word 為 deep 且 window size 設為 1 時，隨機產生的訓練資料可能如下：

Context Center Pair?
like deep True
orange deep False
zebra deep False
NLP deep False

除了第一行以外，其他行都是隨機產生的，至於要產生多少行就要看文本的大小了，小文本的話可以通常會是 5~20 個，大文本的話 1~5 個就好。

Objective Function

我們希望將目標函數$$ J(\theta) = \frac{1}{T}\sum_{t=1}^{T}J_{t}(\theta) $$最大化，其中 $J_{t}(\theta)$ 表示在第 t 個 window 時的目標函數，定義為
$$ J_{t}(\theta) = \log{\sigma(\mu_{o}^{T}\nu_{c}}) + \sum_{i=1}^{k}E_{j～P(w)}[\log{\sigma(-\mu_{j}^{T}\nu_{c})}] \tag{2} $$
整個式子可以分為兩部分， 前半部是熟悉的計算真 (context, center) 組合的相似度，也是我們需要最大化的部分。後半部就是我們隨機產生的 k 個假(context, center)組合，並計算它們的相似度。這一部分我們就要把它最小化。（利用$\sigma(-x) = \sigma(1-x)$）

在隨機產生假 (context, center) 的部分，字彙 w 被選取的機率 $P(w) = \frac{U(w)^{\frac{3}{4}}}{Z}$，表示將會根據字彙的 unigram distribution 並取 $\frac{3}{4}$ 次方，如此可以讓頻率較低的字彙有較高的機會被選擇到。

Count based Word Vector Model

    Word2Vec 需要走過文本內的每一個字，看哪些字彙會與這個字共同出現，藉此預測一個字彙的上下文。那有沒有方法可以直接抓到詞彙共同出現的關係，不需要一個字一個字看呢？有一個方法是建立window based co-occurrence matrix。如文本內只有兩個句子

  • I like NLP
  • I like deep learning

當 window size 為 1 時，則 co-occurrence matrix 如下，矩陣的每一行也就是一個字彙的向量。

Counts I like deep learning NLP
I 0 2 0 0 0
like 2 0 1 0 1
deep 0 1 0 1 0
learning 0 0 1 0 0
NLP 0 1 0 0 0

簡單說一下這個模型的缺點:

  1. 這樣的模型可以想像一個字彙向量會有相當大的維度，因此會需要非常大的儲存空間。
  2. 有些詞彙如停用詞會有非常高的頻率，這也會影響向量的的表示。
  3. 當有新的字彙加入或是有新的文件加入時，整個模型也需要重新計算。

降維的問題可以用 SVD 解決，不過 SVD 的計算複雜度非常高（$O(mn^{2})$ 其中m 為文本內的句子數，n 為字彙數）。而停用詞的高頻問題可以設定頻率的最大值，當頻率超過某一限度則以較小數值表示即可。

Comparison

    最後比較一下 Count-based Model 與 Predictive-based Model 的差別

  • Count-based
    1. 訓練時間快
    2. 善用了整個文本的統計資訊
    3. 主要用來抓住字彙之間的相似度
    4. 高頻詞的影響比較大
  • Predictive
    1. 訓練時間隨著文本增加
    2. 無法善用整個文本的資訊
    3. 可以抓到字彙相似度以外更複雜的關係

GloVe

    Pennington et al.(2014) 提出的 GloVe 就是為了能夠融合上述兩種模型的優點，既能快速訓練，也能應付大量的文本。模型一開始也需要建構 co-occurrence matrix ，而它的目標函數如下：
$$J(\theta) = \frac{1}{2}\sum_{i,j=1}^{W}f(P_{ij})(\mu_{i}^{T}\nu_{j} - \log{P_{ij}})^{2} \tag{3}$$
其中 $P_{ij}$ 為兩個字彙組合的共同出現頻率，$ f(x) $ 則是用於調整高頻字彙組合的函數。整個式子針對每一個有共同出現的字彙組合將 $\mu_{i}^{T}\nu_{j} - \log{P_{ij}}$ 的結果最小化。另外要注意的是，在這個模型中一個字彙仍然會有兩個向量 $\mu$ 與 $\nu$，訓練完成後把兩個向量相加即為最後的向量。

CS224 Lecture2 筆記 | Word Vector Representation

Posted on 2018-06-08

    先附上影片連結以及投影片連結

    表示字彙的方式或是將字彙向量化的方法最直觀的想法可能就是 one-hot 了，不過這樣的表示方法有很多缺點，像是向量維度會非常大，或是沒辦法表現字彙與字彙之間的關係。所以就有人提出了以字彙的前後文來表示的方法。核心的想法就是一個字彙的意思可以由經常出現在它前後的字彙(context)來決定。這也就是 Word2Vec (Mikolov et al., 2013) 核心的想法。

Intuition

  • 我們手上有大量的文本資料
  • 每一個字彙都可以被一個向量表示
  • 文本內的每一個位置都有 center word c 以及多個 context words o
  • 利用 c 的向量與 o 的向量的相似度計算 $P(o|c)$
  • 持續調整向量以最大化 $P(o|c)$

Cost Function

    針對文本上的位置 $t=1, \dots, T$，給定 center word $w_{t}$，預測 context words 在其左右 $m$ 位置的機率模型可以寫成
$$ L(\theta) = \prod_{t=1}^{T} \prod_{-m \leq j \leq m \atop j \neq 0} P(w_{t+j} | w_{t}; \theta) \tag{1}$$ 舉個例子，當我要計算 context word 為 you，center word 為 like 的機率時，可以表示為 $$ P( \mu_{you} | \nu_{like} ) = P( I | like; \mu_{you}, \nu_{like}, \theta ) $$ 其中 $ \theta $ 是所有我們需要optimized的變數。

    而 Cost Function 定義為 $$ J(\theta) = -\frac{1}{T} \log L(\theta) = -\frac{1}{T} \sum_{t=1}^{T} \sum_{-m \leq j \leq m \atop j \neq 0} \log P(w_{t+j} | w_{t}; \theta) \tag{2}$$ 所以最小化 cost function(式2) 相當於最大化我們的預測模型(式1)。

    那該如何計算 $ P(w_{t+j} | w_{t}; \theta) $ 呢？為了 optimize 時方便，我們會先用兩個向量來表示一個字彙 $w$

  • $ \nu_{w} $: 表示當 $w$ 是一個 center word 時的向量
  • $ \mu_{w} $: 表示當 $w$ 是一個 context word 時的向量

    在我們的 case 裡，若文本共有 V 個字彙，那 $\theta$ 會是一個 2 * V * d 維的矩陣，每一個 row 就是一個字彙的 d 維向量，並且一個字彙有兩個向量。當完成訓練後，再將同一字彙的兩向量取平均作為該字彙的向量。所以給定一個 center word c，o 為其 context word 的機率可以表示為
$$ P(o | c) = \frac{exp(\mu_{o}^{T} \nu_{c})}{\sum_{w \in V} exp(\mu_{w}^{T} \nu_{c})} \tag{3}$$ 式子的分子是個內積，相當於在比較 o 和 c 的相似度，內積結果越大代表兩向量越相近 ( 利用 $ \vec{a} \cdot \vec{b} = |\vec{a}||\vec{b}| \cos(\theta$) )。分母的部分是在作 Normalization。整個式子其實是一個在深度學習常用到的函數 softmax ，將一個 d 維度向量內的實數 $ x_{i} $ 轉換成機率分布 $ p_{i} $，並滿足$$ R^d \rightarrow R^d \Big| p_{i} > 0, \sum_{i=1}^{d}p_{i} = 1 $$

Gradient Descent

    Gradient Descent 是一個幫助我們 minimize cost function $J(\theta)$ 的演算法，它主要的想法是根據現值 $\theta$ ，計算 $J(\theta)$ 的gradient，接下來將 $\theta$ 往 -gradient 的方向更新，並重複這動作直到收斂。如 $$ \theta’ = \theta - \alpha \nabla_{\theta} J(\theta) \tag{4}$$ 其中 $\alpha$ 為 learning rate 。寫成 code 大家可能看更懂。

    上面的方法在優化過程中，需要跑過文本中所有的 (center word, context word) 的組合，$\theta$ 才會作一次更新。當字彙量很大時， $\nabla_{\theta} J(\theta)$ 的計算是非常耗時的。一個優化的方法就是Stochastic Gradient Descent啦。

Word2Vec 模型的訓練

Posted on 2018-06-02

  Word2Vec 是自然語言處理上很重要的工具，透過一個 NN 的模型將字彙轉變成向量，轉變成的向量可以拿來作許多有趣的應用。這篇文章將介紹這個將字彙轉變為向量的 NN 模型是如何訓練的，模型的結構、輸入的訓練資料以及輸出又是什麼樣子，會以 Skip-Gram 為例。

Skip-Gram

  詞向量其實是訓練 Skip-Gram Model 產生的附屬品，在 Coursera 上的 Deep Learning 課程裡，Skip-Gram 的介紹是

Given a sentence, randomly pick an word as context word, and randomly pick another word within some window to be the target word.

（window 指的是在 context word 的上下文範圍。如當 window = 2 時，context word 的前後 2 個詞都是其上下文。）
Skip-Gram 模型白話一點的介紹可以說是給定一個字彙(context word)，預測一目標詞(target word)為其上下文的機率。例如當你的 context word 是 「數學」時，你會期望「老師」會有比「麥當勞」更高的機率為「數學」的上下文。而且當這樣的模型訓練好以後，只需要取模型所學習到的權重，這些權重就是我們想要的”Word Vector”。

NN Model

  我們希望建立一個 NN 模型，當模型輸入一個字彙時，會輸出其他字彙為其上下文的機率。一個三層結構的 NN 模型可以達到這個目標。這個模型需要一連串的(context, target)的組合作為訓練資料。如當訓練的 corpus 僅有一句話且window size 為 1 時，訓練資料就如：

Input Layer

  假設我們共有 K 個字彙，因為無法直接將字彙以 string 的形式輸入給 NN ，所以必須將輸入的字彙經過某種轉換。如 one-hot 編碼，將字彙轉換為 K 維的向量後作為輸入資料。

Hidden Layer

  在 Hidden Layer 的 neuron 數量將會決定我們用多少個特徵來表示一個字彙，如在 Hidden Layer 的 neuron 數量是 D ，代表未來每個字彙可以用為 D 維的向量表示。
  Hidden Layer 會接收來自 Input Layer 的 K 維向量，所以可以想像 Hidden Layer 是個 K * D 的矩陣，並且這個矩陣在經過訓練後的權重就是字彙的詞向量。未來當我們以第 i 個字彙的 one-hot 向量乘以這個矩陣時，就相當於拿出這個矩陣的第 i 行向量，所以我們其實是在訓練出 Word Vector 的 lookup table。

Output Layer

  Output Layer 是一個 softmax 的分類器，目的是把每個 neuron 的輸出正規化，並且讓總和為 1 。所以模型最後輸出的是機率的分布。

  以上所講的是一個最簡單的模型，這個模型還存在一些缺點，如字彙量 K 太大時，計算 softmax 是很大的負擔，所以後來還有很多的改進方法，如 Negative Sampling 等，未來再繼續補充。

Dockerfile - CMD vs ENTRYPOINT

Posted on 2018-05-26

老實說，之前無論是寫 Dockerfile 或是 docker-compose.yml，我一直都只有用 CMD 指令。直到幾天前被問到 CMD 和 ENTRYPOINT 的差異時才知道 ENTRYPOINT 的存在…，只好事後趕快惡補一下，以下是我網上搜尋後稍微整理的筆記。關於 CMD 和 ENTRYPOINT 的語法格式，在官方文件以及網路上有相當多的介紹了，這裡就不提及了。

以啟動容器的指令 docker run -i -t ubuntu bash 為例，這個指令可以拆成三部分：

  • 容器啟動時使用的 image 是 ubuntu
  • 啟動後執行預設的ENTRYPOINT (/bin/sh -c)
  • 啟動後執行的指令為 bash

表示容器透過 ENTRYPOINT 的預設指令 /bin/sh -c 來執行 bash指令，所以完整的指令其實是 /bin/sh -c bash。這也說明了為什麼我之前可以完全只用 CMD，因為其實背後已經有預設好的 ENTRYPOINT 了啊！
另外值得一提的是，在 ubuntu 的 Dockerfile 內，其實 CMD 也有預設的指令 CMD ["bash"]，故當你僅執行 docker run -i -t ubuntu 時，會有與前面的例子相同的效果。

在StackOverflow上有一個很好的例子。大家可以試試看在Dockerfile內將 ENTRYPOINT 設定為 /bin/cat，並且執行 docker run ubuntu /etc/passwd，這個例子應該更能幫助理解兩者用法的不同。

最後附上有人整理出關於 Docker 的 RUN, CMD, ENTRYPOINT 三個指令的 Best Practice

  • 使用 RUN 指令來建構容器、安裝套件。
  • 若容器的用途是用來運行某項服務，如MySQL，則優先使用 Exec 格式的 ENTRYPOINT 指令。CMD 指令可為 ENTRYPOINT 提供額外的預設參數，同時也可利用 docker run 來替換預設參數。
  • 若想為容器設置預設的啟動指令，可使用 CMD 指令。使用者可在 docker run 中替換此預設指令。
